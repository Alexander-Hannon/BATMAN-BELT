[IF...ELSE]

IF STATEMENTS

Components:

- if (start statement)
- []; (condition)
- then (execute code)
- fi (end statement)

Ex.)
num=15
if [ $num -gt 10 ]: then 
  echo "Number is greater than 10"
fi


IF ELSE STATEMENTS

Components:

- if (start statement)
- []; (condition)
- then (execute code)
- else (if condition is not true)
- fi (end statement)

Ex.)
num=8
if [ $num -gt 10 ]; then 
  echo "Number is greater than 10"
else
  echo "Number is less than 10"
fi


ELIF STATEMENTS

Components:

- if (start statement)
- []; (condition)
- then (execute code)
- elif (move to next condition if 1st not met)
- []; (second condition)
- then (execute code)
- else (if both conditions not met)
- fi (end statement)

Ex.)
num=10
if [ $num -gt 10 ]; then
  echo "Number is greater than 10"
elif [ $num -eq 10]; then 
  echo "Number is exactly 10"
else
  echo "Number is less than 10"
fi


NESTED IF STATEMENTS
(Place If Statement inside another)

Ex.)
num=5
if [ $num -gt 0 ]; then
  if [ $num -lt 10 ]; then
    echo "Number is between 1 and 9"
  fi
fi

[BASH LOOPS]


FOR LOOPS (Specified # of times)

Components:

- for (begin loop)
- i (index/iteration)
- in (points to range)
- {}; (range)
- do (execute code)
- done (finih loop)

Ex.) 
for i in {1..5}; do
  echo "Iteration $i"
done


WHILE LOOPS (While a specified condition is true)

Components:

- while (begin loop)
- []; (condition)
- do (execute code)
- done (finish loop)

Ex.)
count=1
while [ $count -le 5 ]; do
  echo "Count is $count"
  ((count++))
done


UNTIL LOOPS (Loop Until a condition becomes true)

Components:

- until (begin loop)
- []; (condition)
- do (execute code)
- done (finish loop)


BREAK AND CONTINUE
(Break and Continue statements are used to control loop execution. "break" exits the loop, while "continue" skips the next iteration.)

Ex.)
for i in {1..5}; do
  if [ $i -eq 3 ]; then
    continue
  fi
  echo "Number $i"
  if [ $i -eq 4 ]; then
    break
  fi
done


NESTED LOOPS (Loop within Loop)

Ex.)
for i in {1..3}; do 
  for j in {1..2}; do
    echo "Outer loop $i, Inner loop $j"
  done
done

SCHEDULING TASKS - Cron

(The cron system is a time-based job scheduler in Unix-like operating systems. It automates the execution of tasks (known as cron jobs) at specified intervals. Crontab is used to set up and control cron jobs.)


CRONTAB SYNTAX

crontab [options]

Options:
-e (Edit the crontab file for the current user)
-l (List the crontab entries for the current user)
-r (Remove the crontab file for the current user)


SETTING UP CRON JOBS
(Cron jobs are defined using a specific syntax in the crontab file. Each line in the file represents a task and follows this format:)

* * * * * command_to_execute

* Minute: 0-59
* Hour: 0-23
* Day of Month: 1-31
* Month: 1-12
* Day of Week: 0-7 (0 and 7 are Sunday)

(Each asterisk can be replaced with a specific value or range to schedule the command at specific times.


SCHEDULING A TASK

Ex.)
0 0 * * * /path/to/script.sh

(This entry will execute /path/to/script.sh every day at 00:00 (midnight)

Ex.)

* * * * * /path/to/script.sh

(This entry will execute /path/to/script.sh every minute


COMMON USES

- Automate system maintenance tasks, like backups and updates
- Schedule scripts to run at specific times or intervals
- Perform regular monitoring and reporting tasks
